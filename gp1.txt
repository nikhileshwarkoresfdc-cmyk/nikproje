1. Data Modeling & Architecture
1. How did you design a scalable data model for a high-volume org?

Answer:
In one project handling ~5M+ records, I avoided lookup skew by:

Using indexed fields (External IDs)
Avoiding too many lookups to a single parent
Splitting objects using vertical partitioning
Using Big Objects for historical data
2. How do you handle lookup skew?

Answer:

Avoid more than 10k records pointing to one parent
Use ownership distribution
Replace lookup with junction object
Use Queue-based ownership
3. When do you use Master-Detail vs Lookup?

Answer:

Use Master-Detail when:
Tight coupling
Roll-up summaries needed
Use Lookup when:
Loose relationship
Independent ownership/security
